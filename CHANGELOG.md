# Changelog

## 2026-05-05

### Added

- 新增桌面端主界面 Dashboard：
  - 应用启动后先进入“小红书 Agent 工作台”
  - 主界面提供小红书登录状态、AI 设置状态、任务执行入口、每日报告入口、帖子仓库入口 5 个入口卡片
  - 主界面同步显示当前任务模式、运行状态与帖子仓库数量
- 新增登录弹窗：
  - 小红书登录模块从任务执行页移出
  - 主界面“开始登录”打开登录弹窗
  - 弹窗复用原登录状态、打开登录浏览器、召回/移出浏览器、删除账号、截图预览能力
- 新增小红书账号池：
  - 主界面登录入口改为“账号管理”
  - 登录弹窗新增账号池 UI，支持输入账号用户名并生成 `xhs_browser_data_{用户名}` 浏览器数据目录
  - 支持账号池列表、设为当前、登录/重新登录、删除账号
  - 账号池数据持久化到 SQLite，不再使用独立 JSON 文件
  - 任务执行页顶部新增“执行账号”下拉，可为搜索任务选择当前账号
- 新增竞品监控入口：
  - 主界面新增“竞品监控入口”卡片
  - 新增独立空页面承载后续竞品监控能力
- 新增项目文档：
  - `BACKEND_CHANGELOG.md` 记录 Electron 主进程、Python 运行时、SQLite 存储和搜索核心变化
  - `PROJECT_STRUCTURE.md` 说明当前目录结构、关键入口、运行时数据文件和推荐启动方式
- 新增每日任务链路：
  - 主页支持”立即执行 / 每日任务”模式切换
  - 支持保存每日任务名称、执行时间、启用状态
  - 新增 `Daily Draft` 草稿卡片预览每日任务状态
  - 新增主进程定时轮询调度器
  - 新增每日任务运行记录与 SQLite 持久化字段
- 新增每日任务主进程服务：
  - `electron/main/services/scheduledTaskService.js`
  - `electron/main/services/scheduledTaskStoreService.js`
- 新增帖子仓库以外的每日任务持久化能力：
  - `desktop_runtime/post_store/` 补充 scheduled task / scheduled run 的存储与 CLI 入口
- 新增每日报告功能：
  - 报告入口放置在”帖子仓库”按钮旁边
  - 使用 SQLite 存储报告数据（`daily_reports` 和 `report_posts` 表）
  - 每日任务完成后自动生成报告
  - 报告包含 LLM 生成的专业分析内容
  - 支持查看报告详情（Markdown 格式渲染）
  - 支持从报告跳转到相关帖子
  - 报告列表采用响应式网格布局（默认 5 列，自适应 4/3/2/1 列）
- 新增报告生成模块：
  - `desktop_runtime/post_store/report_generator.py` 负责 LLM 提示词设计和数据准备
  - 支持解析小红书互动数据格式（”1.2k”、”5.6w” 等）
  - 使用 OpenAI 兼容接口生成报告内容
- 新增报告相关 IPC 通道：
  - `report:list` - 获取报告列表
  - `report:get` - 获取单个报告详情
  - `report:get-posts` - 获取报告关联的帖子
- 新增报告前端控制器：
  - `electron/renderer/scripts/reportController.js`
  - 支持报告列表渲染、详情查看、帖子跳转
  - 支持章节卡片化的 Markdown 渲染

### Changed

- 桌面端页面层级调整：
  - 主界面作为默认入口
  - 原任务界面改为“任务执行”子页面
  - 帖子仓库与每日报告关闭后返回主界面
  - 任务记录侧栏只在任务执行页显示
- AI 设置入口调整：
  - 从任务执行页移到主界面 AI 设置卡片
  - 保持全局 OpenAI 兼容接口配置语义
- 小红书登录入口调整：
  - 任务执行页不再展示登录模块
  - 主界面保留登录状态卡片与登录弹窗入口
  - 主界面移除“召回浏览器”快捷按钮
  - 主界面登录按钮统一为“账号管理”
  - 登录弹窗继续保留浏览器召回/移出能力
- 小红书登录态目录调整：
  - 不再使用固定单账号 `xhs_browser_data/` 作为唯一登录态
  - 每个账号使用独立 `xhs_browser_data_{用户名}/` 目录
  - 账号池允许为空，必须先新增账号再登录
- 搜索执行账号调整：
  - 搜索 worker 跟随任务执行页顶部选择的当前账号
  - Electron 将当前账号浏览器目录传给 Python runner
  - Python runner 不再固定读取根目录 `xhs_browser_data/`
- 任务输入按钮布局统一：
  - “立即执行”模式下的“启动搜索 / 停止”右对齐
  - 与“每日任务”模式下的“保存每日任务”对齐方式保持一致
- 主页主工作区重新编排：
  - 左侧为任务输入与每日任务配置
  - 右侧改为”运行状态”叠放”内部日志”
  - 同排卡片按桌面宽屏保持等高
- 每日任务交互调整：
  - 保存按钮在每日任务模式下右对齐
  - `Daily Draft` 卡片压缩为更紧凑的信息摘要
  - 中小窗口下提前解除固定高度，避免左侧卡片被裁切
- 右侧日志区交互调整：
  - “运行状态”的详细过程改为面板内滚动
  - “内部日志”改为面板内滚动
  - 不再把整页 UI 一直向下顶
- 每日任务触发判定调整：
  - 不再只按日期去重
  - 改为按”日期 + 时分”的调度槽位判断是否已触发
- 搜索运行态诊断增强：
  - 调度器跳过日志补充 `phase` 与运行时长
  - 便于区分预检、worker 启动、真实运行阶段
- 任务启动前的登录心跳检测已移除：
  - 不再在启动搜索前主动探测登录态
  - 避免再次干扰或挤掉本地小红书登录状态
- 软件主窗口默认尺寸调整：
  - 从 `1280x720` 调整为 `1280x960`
  - 最小尺寸从 `1100x680` 调整为 `1100x820`
  - 窗口比例更接近 4:3
- 报告页面交互调整：
  - 进入报告页面时自动隐藏任务记录侧栏
  - 关闭报告页面时恢复任务记录侧栏显示
- 报告详情样式优化：
  - 报告内容按章节卡片化展示
  - 每个章节独立卡片，带渐变背景和阴影
  - 章节标题带下划线分隔，使用项目主题色
  - 列表项使用自定义红色圆点
  - 加粗文字带浅色背景高亮
  - 数字用金色标签 + 等宽字体显示
  - 优化行间距和段落间距
  - 支持代码块、斜体、分隔线等 Markdown 元素
- 帖子仓库筛选状态管理优化：
  - 新增 `state.fullLibraryResults` 保存完整帖子列表
  - 从报告跳转到帖子仓库时临时筛选报告相关帖子
  - 直接点击”帖子仓库”按钮时恢复完整列表
  - 修复从报告查看帖子后无法返回完整列表的问题
- 搜索完成后结果处理优化：
  - 新结果合并到现有预览区，而非替换
  - 确保手动搜索和每日任务的结果都能正确显示
- 登录截图预览优化：
  - 从整窗截图改为优先裁剪二维码区域
  - 使用 PNG 截图，避免二维码因压缩和缩放难以扫码
  - 前端预览避免强制缩小整张浏览器截图

### Fixed

- 修复任务执行页承载过多全局入口导致页面职责混乱的问题：
  - 登录与 AI 配置改为主界面全局入口
  - 任务执行页聚焦搜索需求、每日任务配置、运行状态和结果预览
- 修复主界面首次进入时任务记录侧栏可能闪现的问题：
  - 任务记录侧栏初始状态改为隐藏
  - 进入任务执行页时再显示侧栏入口
- 修复每日任务已保存但在到点后未真正触发的问题：
  - 调度器启动逻辑补到实际生效的 Electron 入口 `electron/main.js`
  - 不再只写在未被 `npm start` 使用的备用启动链路中
- 修复每日任务”保存后立即检查”与轮询触发在同一天改时间后仍被误判为已执行的问题。
- 修复每日任务触发后 `activeRun` 卡住导致后续轮询始终跳过的问题：
  - Python 子进程启动异常时立即收口
  - 调度运行记录能正常回写完成态
- 修复 `Daily Draft` 卡片在较小分辨率下被遮挡、被下方内容顶住的问题。
- 修复 `Daily Draft` 卡片在失败摘要过长时把布局向右撑爆的问题：
  - 失败态不再直接展示整段底层报错
  - 卡片标题、元信息、摘要文案补充强制断行
- 修复右侧日志卡片未形成内部滚动、仍持续推高页面的问题。
- 修复报告生成时阻塞主进程的问题：
  - 报告生成改为异步执行，不阻塞 UI
  - 使用 `spawnPythonModule` 替代同步调用
- 修复 preload 未暴露报告相关方法导致前端调用失败的问题：
  - 补充 `listReports`、`getReport`、`getReportPosts` 到 `contextBridge`
- 修复报告服务缺少 `execPythonModule` 导入的问题。
- 修复报告页面点击”查看相关帖子”时调用不存在函数的问题：
  - 将 `renderLibraryPage()` 改为正确的 `renderResults()`
- 修复从报告查看帖子后再点击”帖子仓库”按钮永远只显示报告帖子的问题：
  - 新增完整列表备份机制
  - 正常打开帖子仓库时恢复完整列表
- 修复保存每日任务时立即触发执行的严重问题：
  - 移除 `saveDailyTask` 中的立即触发逻辑
  - 保存后等待调度器在正确时间点触发
  - 避免在非预期时间（如 13:04 保存 10:00 的任务）误触发
- 修复每日任务完成后结果预览和报告消失的严重问题：
  - `handleCompleted` 改为合并新结果到现有预览，而非替换
  - 确保搜索完成后预览区能正确显示所有结果
  - 帖子仓库始终保持完整列表
- 修复登录二维码预览因整窗缩放导致扫码困难的问题：
  - 截图裁剪围绕二维码和扫码提示区域
  - 调整裁剪横向偏移，避免二维码边缘被遮挡
- 修复多账号登录态无法区分的问题：
  - 账号目录加入用户名后缀
  - 搜索任务按当前账号目录启动浏览器
