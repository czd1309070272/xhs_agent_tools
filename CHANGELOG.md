# Changelog

## 2026-05-04

### Added

- 新增 Electron 桌面壳，包含主进程、预加载桥和渲染层 UI。
- 新增桌面搜索桥接脚本 `desktop_search_runner.py`：
  - 复用 `xhs_agent_tool.py` 的搜索逻辑
  - 去掉桌面链路里的登录检查
  - 通过 JSON 事件流向 Electron 回传日志、状态和结果
  - 搜索浏览器默认保持可见，便于调试
- 新增桌面登录工作流：
  - 启动时检查 `xhs_browser_data/`
  - 未检测到本地浏览器数据时提示先登录
  - 支持一键启动小红书登录浏览器
- 新增登录预览面板：
  - 浏览器截图回显到桌面 UI
  - 前端显示“每 5 秒刷新一次”的倒计时提示
- 新增登录浏览器窗口控制：
  - 默认以 `1280x720` 固定分辨率启动
  - 默认直接移到屏幕外
  - 支持一键召回到可视范围
  - 支持再次移回屏幕外
- 新增账号清理能力：
  - 登录面板提供“删除账号”按钮
  - 可清理本地 `xhs_browser_data/`
- 新增结果详情弹层：
  - 点击结果卡片可展开完整正文
  - 支持查看标签、统计信息和原帖跳转
- 新增图片展示能力：
  - 结果卡片显示首图缩略图
  - 详情弹层提供多图轮播
  - 支持应用内放大查看图片
- 新增独立“帖子仓库”页面：
  - 从主页结果预览区一键进入
  - 复用现有结果卡片样式展示已搜到的帖子
  - 支持从仓库页继续打开帖子详情弹层
- 新增左侧任务记录侧栏：
  - 保存运行过的任务输入
  - 保存每次返回的帖子数量
  - 保存任务完成时间和耗时
  - 记录持久化到本地 `task_run_history.json`
- 新增本地帖子存储模块 `post_store.py`：
  - 使用 SQLite 管理帖子主数据
  - 存储帖子、图片和标签
  - 提供给桌面端读取帖子仓库
- 新增主页结果预览工具按钮：
  - “帖子仓库”
  - “清理预览”
- 新增 Electron 依赖与启动脚本：
  - `electron`
  - `playwright`
  - `npm start`
- 新增帖子仓库管理工具栏：
  - 关键词搜索
  - 默认排序
  - 时间正序 / 倒序
  - 点赞最多
  - 收藏最多
  - 评论最多
- 新增帖子仓库删除能力：
  - 单篇删除
  - 批量删除
  - 删除后同步更新主页预览态
- 新增内部诊断日志面板：
  - 独立展示主进程内部状态
  - 用于排查 preload / renderer / SQLite 读取链路异常
- 新增主页 AI 设置能力：
  - 支持配置 OpenAI 兼容接口的 Base URL
  - 支持配置 API Key
  - 支持配置模型名称
  - 本地配置保存到 `ai_settings.json`

### Changed

- 去掉 Electron 默认菜单栏。
- Electron 的“启动搜索”按钮已从演示模式切到真实 Python 搜索 worker。
- 软件主窗口默认尺寸调整为 `1280x720`。
- 桌面 UI 移除手动筛选项：
  - 目标条数
  - 排序偏好
  - 时间范围
- 桌面 UI 移除右侧 `Desktop Preview` 装饰区，主工作区改为全宽布局。
- 结果详情布局改为响应式：
  - 普通窗口采用上下结构
  - 宽屏/全屏时采用左图右文结构
- 结果详情中的标题改为居中显示。
- 点赞、评论、收藏、作者信息移到详情底部。
- 登录预览从高频更新改为 5 秒一次。
- 登录预览采集策略从依赖 `screencast` 推帧，调整为主进程主动抓图：
  - 首帧更快
  - 避免页面静止时倒计时卡在 `0 秒`
- 结果区交互调整：
  - “清理结果”语义收紧为“清理预览”
  - 清理动作只清空主页预览，不影响帖子仓库
  - 帖子仓库改为独立页面切换，不再覆盖在主页上方
- 结果卡片布局压缩为更紧凑的 4 列展示：
  - 主页结果预览默认一排 4 个
  - 帖子仓库默认一排 4 个
  - 卡片尺寸、标题和间距同步收紧
- 页面滚动体验调整：
  - 主页和帖子仓库的默认右侧滚动条隐藏
  - 页面仍保留滚动能力
  - 打开帖子仓库时默认回到顶部
- 任务记录入口交互调整：
  - 从独立按钮改为贴边“书签页”式标签
  - 标签跟随任务记录面板一起展开/收起
  - 标签位置调整到左侧中部，避免遮挡顶部标题
- 帖子存储链路调整：
  - 搜索结果改为优先写入本地 SQLite
  - `xhs_agent_result.json` 继续保留，作为导出和调试输出
  - Electron 启动时优先从 SQLite 读取帖子仓库，失败时回退到 JSON
- 帖子仓库交互调整：
  - “选择”改为更轻量的 pill 式按钮
  - 选择当前帖子时不再触发整页重渲染
  - 进入帖子仓库时自动隐藏左侧任务记录侧栏
- 桌面端信息密度调整：
  - 全局面板、按钮、输入框和日志项整体收紧
  - 登录预览、结果卡片、详情弹层同步缩小
  - 宽屏下结果区默认提升为 5 列展示
  - 中等宽度窗口按 4 / 3 / 2 列自适应切换
- Python 运行入口调整：
  - `post_store.py` 与 `desktop_search_runner.py` 的职责已拆入 `desktop_runtime/`
  - Electron 不再依赖根目录空壳脚本路径
  - 主进程改为通过 `python -m` 调用模块入口
  - 存储入口改为 `desktop_runtime.post_store.cli`
  - 搜索 runner 入口改为 `desktop_runtime.search_runner.bootstrap`
- Electron 启动链路调整：
  - 保留模块化服务拆分
  - `main.js` 与 `preload.js` 恢复为扁平入口
  - 优先保证 `desktopApi` 注入稳定，再复用内部模块
- LLM 配置链路调整：
  - 桌面搜索 worker 改为使用通用 OpenAI 兼容接口
  - 保留 `DEEPSEEK_*` 环境变量兼容兜底
- Python 搜索核心结构调整：
  - 原高耦合的 `xhs_agent_tool.py` 已拆分为 `xhs_agent/` 包
  - 浏览器、意图、打分、编排、CLI 入口已按职责分离
  - 桌面端与命令行端已复用同一套搜索主流程
- 命令行包装脚本位置调整：
  - 根目录包装脚本已移动到 `scripts/`
  - 命令行搜索入口改为 `python -m scripts.xhs_agent_tool`
- 搜索意图链路调整：
  - 拆分为“意图分类”与“搜索计划”两段 LLM 调用
  - 补充视觉型需求默认“最多点赞”与“多图优先”规则
  - “美女图”等偏视觉需求默认更依赖互动数据与图片数量代理信号
- 搜索探索策略调整：
  - 首轮点击名额不再固定为 `min(3, still_need)`
  - 改为由 LLM 返回 `click_budget`，再由代码按需求类型限幅
  - 视觉型 / 多图型需求允许更高点击预算，但仍保留硬上限控风控

### Fixed

- 修复登录截图偶尔显示到错误区域的问题，强制限制在登录预览框内部。
- 修复浏览器启动后闪烁问题：
  - 固定浏览器分辨率
  - 收紧自动点击登录入口的逻辑
  - 降低预览刷新频率
- 修复登录预览倒计时体验：
  - 无首帧时显示“首帧等待中”
  - 超过刷新周期时显示“正在获取下一张截图...”
- 修复结果图片区交互：
  - 单图时隐藏前后切换按钮
  - 第一张图隐藏“前一张”
  - 最后一张图隐藏“后一张”
- 修复放大图片体验：
  - 图片点击后改为应用内放大，不再直接跳外部网页
  - 放大层面板背景改为透明
- 隐藏结果详情右侧默认滚动条，保留滚动能力
- 修复“清理结果”误伤持久化结果文件的问题，改为仅清理当前预览态
- 修复帖子仓库初版以弹窗方式覆盖主页的问题，改为真正的独立页面
- 修复打开帖子仓库时滚动位置停留在中部的问题，进入时强制回到顶部
- 修复任务记录入口遮挡“小红书登录”标题的问题，改为左侧中部贴边标签
- 修复任务记录侧栏收起后仍露出部分面板内容的问题，收起时仅保留标签可见
- 修复帖子主存储只能依赖单个 JSON 文件的问题，补上 SQLite 本地持久化层
- 修复“清理预览”后主页结果再次被 SQLite 回填的问题，改为单独维护 `preview_results.json`
- 修复帖子仓库启动后偶尔读不到 `xhs_agent_data.db` 的问题，恢复可工作的 Electron 主进程 / 预加载入口链路
- 修复 preload 未注入导致桌面按钮全部失效的问题：
  - “打开登录浏览器”无响应
  - “启动搜索”无响应
  - 内部日志面板无内容
- 修复帖子仓库选择时的闪烁问题，改为局部更新选择态而非整页重绘
- 修复删除根目录 `desktop_search_runner.py` 后桌面端无法启动搜索的问题，改为稳定依赖包模块入口而非脚本文件路径
- 修复包内 `cli` / `bootstrap` 直接模块执行时未真正进入 `main()` 的问题
- 修复 `python -m` 执行时因包初始化副作用导致的模块入口冲突问题
- 修复用户明确指定条数时仍可能被 LLM 提前判定完成的问题：
  - 未达到目标数量时忽略 `satisfied=true`
  - 最少保证继续进入下一轮搜索
  - 缺少 `next_keyword` 时使用当前关键词兜底续搜
- 修复搜索结果与数据库历史标题重复时仍被收录的问题：
  - LLM 接受后新增数据库标题去重
  - 与本次运行已接受标题一并比对
  - 若本轮因重复被过滤导致数量不足，则强制继续下一轮
